# Status Bar + Adaptive Refresh Cadence — Design

**Date:** 2026-06-18
**Status:** Approved (design); pending spec review

## Goal

Give the Slime Pet a top status bar (time · date · weather · battery · WiFi) and a
power-aware refresh cadence: repaint every 5 minutes on USB, every 15 minutes on
battery, and disable refresh entirely in "sleep mode" (darkness) until light returns.

## Motivation

Today `code.py` fetches time and weather **once at boot** and only repaints the E-Ink
panel on a pose change, a deliberate boop, or a 6-hour scheduled tick. The panel can
therefore show stale time/weather for hours. The user wants the panel to stay current
on a predictable cadence and to carry an at-a-glance status strip, while preserving the
project's calm, offline-first, never-punishing philosophy and battery life.

## Decisions (locked)

- **Layout:** Split bar. Left group = time + date; right group = temp + weather glyph,
  battery %, WiFi glyph.
- **Formats:** 12-hour time (`3:45 PM`), short date (`Jun 18`), °F temperature (`102°`),
  battery percentage (`84%`), WiFi as a drawn bars glyph with a distinct disconnected
  state.
- **Cadence:** USB awake = 300 s; battery awake = 900 s; sleep mode = refresh disabled
  until light returns.
- **Sleep mode entry:** driven by the ambient light sensor with hysteresis.
- **Battery, in the dark:** wake on motion **plus** a quiet 1-hour light-check timer, so
  daylight resumes the pet within an hour without requiring a touch.
- **NeoPixels in sleep mode:** a very dim, slow sleep-breath stays alive (living-heartbeat
  philosophy), not fully off.
- **WiFi glyph semantics:** "live data / online" — connected icon when this cycle's NTP +
  oracle fetch succeeded, disconnected icon when running on cached data.

## Architecture

The project's pure-logic-vs-hardware-adapter split is preserved. All formatting and the
cadence/sleep decisions live in **pure modules** (host-tested with pytest, no hardware
imports). The display adapter and `code.py` consume those decisions.

### New pure module: `slime/statusbar.py`

Pure functions, fully unit-tested:

- `clock_12h(epoch, tz) -> str` — `"3:45 PM"`. Built on the new `timekeeping.hms_from_epoch`.
  Midnight → `"12:00 AM"`, noon → `"12:00 PM"`, minutes zero-padded.
- `short_date(epoch, tz) -> str` — `"Jun 18"`, using a month-abbreviation table.
- `temp_str(oracle) -> str` — converts the oracle's `temp_c` to whole °F: `round(c * 9/5 + 32)`
  then `f"{f}°"`. Returns `""` when `oracle is None`.
- `battery_str(frac) -> str` — `f"{round(frac * 100)}%"`, clamped to 0–100.
- `weather_icon(oracle) -> int | None` — maps the oracle's `weather_tag` to a tile index in
  `statusicons.bmp`; `None` when unknown.
- `WIFI_LIVE`, `WIFI_STALE` — integer constants naming WiFi tile indices.
- `refresh_interval(on_usb, sleeping) -> float | None` — `None` when `sleeping`, else
  `300.0` on USB, `900.0` on battery.
- `is_sleep_mode(light, currently_sleeping) -> bool` — hysteresis: when not currently
  sleeping, enter sleep only if `light < 0.08`; when already sleeping, stay asleep until
  `light > 0.15`. The dead band prevents flicker at the threshold.

Constants: `_SLEEP_ENTER = 0.08`, `_SLEEP_EXIT = 0.15`, `_USB_REFRESH = 300.0`,
`_BATTERY_REFRESH = 900.0`.

### Extended pure module: `slime/timekeeping.py`

- `hms_from_epoch(epoch, tz_offset_hours) -> (h, m, s)` — local hours/minutes/seconds.
  Derives seconds-into-day from `_local_seconds`, independent of the civil-date math.
- `MONTH_ABBR` — `("Jan", ..., "Dec")` table consumed by `statusbar.short_date`.

No behavior change to existing functions.

### New artwork: `assets/statusicons.bmp`

A small indexed-color tile sheet generated by `assets/make_assets.py` (same generator that
builds `slime.bmp` / `accents.bmp`). Tiles:

- Weather glyphs: clear, cloud, rain, storm, heat, moon (indices addressed by
  `statusbar.weather_icon`).
- WiFi connected (`WIFI_LIVE`) and WiFi disconnected/slash (`WIFI_STALE`).

`terminalio.FONT` is ASCII-only, so the weather symbol and WiFi bars cannot be text — they
are drawn bitmaps. Temperature, battery %, time, and date remain text labels.

### Adapter change: `slime/display.py`

A 16 px top strip spanning the full width, with a 1 px divider near y = 17:

- **Left group** (anchored top-left): time label, then date label.
- **Right group** (anchored top-right, laid out right-to-left): WiFi icon tile, battery %
  label, weather icon tile, temp label.
- **Body:** the sprite shifts down to begin just below the bar. The sprite is currently
  scale 2 (64 px → 128 px, full panel height); below a 16 px bar it accepts a small
  bottom clip, with the exact y-offset tuned on-device. The previous top-right
  seasonal/moon accent is removed from the body — weather/moon now live in the bar's
  weather glyph. The quip moves to the lower-right of the body.

`render_frame(frame_index, quip_text="", *, time_str=None, date_str=None, temp_str=None,
battery_str=None, weather_icon=None, wifi_state=None)` — all status fields optional so the
panel still renders offline-first. Unknown time → `--:--`; unknown date → blank; unknown
temp/battery → blank; `wifi_state=None` hides the WiFi tile.

`render_sleep(frame_index, time_str=None, date_str=None, weather_icon=None,
wifi_state=None)` — paints the sleeping creature once with a frozen bar. After it is
called, the loop stops repainting until light returns.

The existing `render_journal` / `render_dream` full-screen takeovers are unchanged (they
are momentary and intentionally hide the bar).

### Orchestration change: `code.py`

**USB continuous loop:**
- Track `last_scheduled_refresh` and `last_ntp_sync` (monotonic timestamps).
- Each iteration, compute `sleeping = statusbar.is_sleep_mode(inputs.light, sleeping)`.
- If `sleeping` just became true → call `display.render_sleep(...)` once; suppress periodic
  repaints while it stays true. NeoPixels switch to a dim slow sleep-breath.
- If `sleeping` just became false → resume: re-fetch oracle, re-render the full frame.
- While awake, every `statusbar.refresh_interval(True, False)` (300 s): re-fetch the oracle,
  re-render the frame with live time/date/temp/battery/WiFi. Re-sync NTP at most hourly
  (`_NTP_RESYNC = 3600.0`) to correct monotonic drift, not on every tick.
- Event-driven repaints (pose change, boop) keep working via the existing `should_refresh`
  path, with `scheduled_interval` set to the active refresh interval.

**Battery wake-cycle path:**
- On wake, read light. Compute `sleeping`.
- **Awake (light):** sync NTP + fetch oracle (battery boot now spends WiFi for live data),
  render the full frame with the bar, then `power.nap(900)` (motion + 15-min timer).
- **Dark (sleep):** render the sleeping frame once (or skip the repaint if already on it),
  then nap on **motion + a 1-hour safety timer** so the next wake re-checks light. If still
  dark on that wake, re-sleep silently (no E-Ink repaint); if light, do a full refresh and
  return to the 900 s cadence.

**WiFi state:** after the time/oracle fetches, set `wifi_state = WIFI_LIVE` if either
succeeded this cycle, else `WIFI_STALE`.

`power.nap` already supports a time alarm + motion pin alarm; the 900 s and 3600 s values
are passed by `code.py`. A small helper may be added for "motion-only-ish with long safety
timer" but no new wake source is introduced (the MagTag has no light-based deep-sleep
wake — hence the safety timer).

## Data Flow

```
sensors.light ─┐
               ├─> statusbar.is_sleep_mode ─> sleeping?
prev sleeping ─┘                                  │
                                                  ├─ yes ─> display.render_sleep (once) + dim breath
on_usb ─────────> statusbar.refresh_interval ─────┘
                                                  └─ no, interval elapsed ─>
   netoracle.fetch ─> oracle.parse ─> oracle (temp_c, weather_tag, moon, ...)
   nettime.sync   ─> synced_epoch ──> timekeeping.now_epoch ─> epoch
        │                                   │
        └─ success? ─> wifi_state           ├─> statusbar.clock_12h ─> time_str
                                            └─> statusbar.short_date ─> date_str
   oracle ─> statusbar.temp_str/weather_icon ; sensors.battery ─> statusbar.battery_str
        └────────────────> display.render_frame(frame, quip, time_str, date_str,
                                                 temp_str, battery_str, weather_icon, wifi_state)
```

## Error Handling / Offline-First

- Any field whose source is unavailable renders blank/placeholder; the creature never
  blocks on the network. No NTP → `--:--` and `WIFI_STALE`; no oracle → cached weather or
  blank temp/icon, `WIFI_STALE`.
- Network fetches stay wrapped so a failure degrades to cache, never a traceback (existing
  `_load_oracle` / `_sync_time` behavior is reused).
- Light-sensor read failure (sensors is `None`) defaults `light` to 0.5 → treated as awake,
  so a dead sensor never traps the pet in permanent sleep.

## Testing

**Pure unit tests (CI `check` job, host pytest):**
- `statusbar`: AM/PM and midnight/noon for `clock_12h`; `short_date` month names and a
  boundary day; °C→°F rounding incl. negative/zero; `battery_str` clamping and rounding;
  `refresh_interval` for all three states; `is_sleep_mode` hysteresis at 0.08 / 0.15 and
  inside the dead band (both directions).
- `timekeeping`: `hms_from_epoch` at known epochs (incl. a tz offset), `MONTH_ABBR` length.
- The existing module scan test (`._replace(` guard) continues to cover the new pure
  modules, which stay hardware-free.

**On-device visual verification (web testing rules):**
- Bar legibility and alignment at the real terminalio font; sprite bottom-clip acceptable;
  weather + WiFi + battery glyphs render; `--:--`/blank fallbacks when offline.
- Cadence: 5-min repaint on USB; 15-min on battery; sleep-mode freeze in the dark and
  resume on light; dim sleep-breath; battery-dark 1-hour light-check resume.

## Scope Guard (YAGNI)

- **No new NVM fields.** Time and weather are derived live each cycle; sleep state is
  computed from the current light reading, not persisted. State stays at v3.
- **No server changes.** The oracle already returns `temp_c` and `weather_tag`.
- **No new wake hardware/source.** The 1-hour safety timer covers light-return on battery.
- **No charging/plug indicator.** Battery % is shown as-is on both USB and battery; a
  dedicated charging glyph is out of scope for this pass.
