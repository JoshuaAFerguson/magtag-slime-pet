# Visitors (Phase 2c-iv) — Design

**Date:** 2026-06-21
**Status:** Approved (scope chosen by user); pending plan

## Goal

Occasionally a small guest creature wanders into the pet's world — summoned by the current
conditions (weather / moon / season / bond) and a rare roll — announced by its own quip, a corner
glyph on the E-Ink screen, a gentle NeoPixel + piezo cue, and a journal note; and it leaves a
keepsake the pet collects forever. Fully offline. This is the final sub-project of Phase 2c.

## Motivation

The pet already reacts to weather, the moon, the seasons, and the bond. Visitors make that world
feel inhabited: a full moon brings an owl, rain brings a snail, deep winter brings a fox. Each
visit is a small ambient delight, recorded as a collected keepsake — the last layer of "this
creature has a life of its own" before Phase 3's hardware work. It stays offline-first and
privacy-first: visitors are derived purely on-device from conditions the pet already knows.

## Decisions (locked)

- **Trigger: offline conditions + a rare roll.** Eligibility is a pure predicate over the current
  oracle/season/bond; a low-probability roll then decides whether an eligible visitor actually
  comes. No network.
- **Effect: quip + journal note + light/sound cue + a keepsake.** A visit shows a special quip and
  a corner glyph, plays a short motif and a NeoPixel flash, adds a journal line, and sets the
  visitor's bit in a persistent NVM keepsake bitmask (collected forever, never cleared).
- **On-screen: a small corner visitor glyph.** A new 12×12 tile per visitor, drawn in a free screen
  corner during the visit (matches the "busy/playful E-Ink is welcome" preference).

## Architecture

Pure logic + device adapters + one NVM version bump + new asset art.

### `slime/visitors.py` (new, pure)

A fixed roster of 8 visitors fills one byte (mirrors the dream-artifacts pattern). Each entry is
`(bit, key, name, quip, glyph_index, condition)` where `condition(oracle, season, tier) -> bool` is
a pure predicate tolerant of `None` oracle/season:

| key | bit | condition | quip |
|-----|-----|-----------|------|
| `owl` | `0b00000001` | `oracle.moon_phase == 4` (full moon) | "an owl settled on the sill" |
| `snail` | `0b00000010` | `oracle.weather_tag in ("rain", "monsoon")` | "a snail came in from the rain" |
| `firefly` | `0b00000100` | `oracle.weather_tag == "extreme_heat"` | "a firefly blinked hello" |
| `moth` | `0b00001000` | `oracle.moon_phase == 0` (new moon) | "a moth circled the lamp" |
| `fox` | `0b00010000` | `season == "winter"` | "a fox passed through the cold" |
| `beetle` | `0b00100000` | `season == "autumn"` | "a beetle trundled by" |
| `sparrow` | `0b01000000` | `season == "spring"` | "a sparrow stopped to rest" |
| `cat` | `0b10000000` | `tier >= 3` (deeply bonded) | "a neighbor's cat curled up" |

Conditions read only `oracle.weather_tag` / `oracle.moon_phase` (the oracle's existing fields), the
`season` string, and the friendship `tier`. The roster is intentionally clean and well-separated:
four oracle-driven (owl, snail, firefly, moth), three season-driven (fox, beetle, sparrow), and one
bond-driven (cat).

Functions (all pure):
- `eligible_visitors(oracle, season, tier) -> tuple[str]` — keys whose `condition` holds. Returns
  `()` when nothing is eligible (e.g. no oracle and no season and a low tier — though `cat` keys off
  tier alone, so a bonded pet always has at least the cat eligible).
- `pick_visitor(keys, choice) -> str | None` — `choice(keys)` or `None` when `keys` is empty.
- `collect(mask, bit) -> int` — `mask | bit` (never clears).
- `name(key)`, `quip(key)`, `glyph(key)`, `bit(key)` — roster lookups.

The probability roll lives in `code.py` (like the memory-quip roll), not here — `visitors.py` stays
purely about *which* visitors are possible and *picking* one.

### State v5 + persistence v5

Add a `visitors` int (collected-keepsake bitmask) as the last `State` field, with v4→v5 migration
defaulting it to 0 — exactly the pattern used for `milestones` in 2c-ii. `_FORMAT_V5 = "<B5ffffIfIIIII"`
(v4 + one trailing `I`); `BLOB_SIZE = _SIZE_V5`. `unpack` gains a v5 branch and the v4/v3/v2/v1
branches pass `0` for `visitors`. The journal-ring start `((BLOB_SIZE//16)+1)*16` is re-verified to
still clear the larger blob (v5 is 61 bytes; `61//16 == 3`, so the ring start stays 64 — unchanged).
`default_state` includes `visitors=0`; `evolve` carries `visitors`.

### Assets — `assets/visitors.bmp`

A new indexed-palette sheet of 8 small (12×12) creature glyph tiles generated by `make_assets.py`
(PIL), one per roster key in `glyph_index` order, same 4-level palette as the existing sheets. Each
glyph is a simple, legible silhouette (owl, snail, firefly, moth, fox, beetle, sparrow, cat).

### `slime/display.py`

`render_frame` gains an optional `visitor_icon=None` kwarg. When set (an int tile index), it blits
the corresponding `visitors.bmp` tile into a free screen corner (below the status bar, clear of the
quip line) during the visit. Drawing is guarded exactly like the existing status tiles — a missing
sheet or bad index never crashes the render.

### `slime/journal.py`

Add a `0b1000` flag bit meaning "a visitor came." `generate_entry` appends a closing
"a visitor came by" line when the bit is set (after the existing presence/busy/inbox lines). The
existing flag bits (`0b1` visited, `0b10` busy, `0b100` inbox) are unchanged.

### `code.py`

After the oracle load + milestone evaluation each cycle (boot and each scheduled refresh), a rare
roll (`_choice` over a weighted tuple, ~1 in 6 — the same idiom as the memory quip) decides whether
to summon a visitor: `pick_visitor(eligible_visitors(oracle, season, tier), _choice)` when the roll
fires, else `None`. On a visit:
- `state = evolve(state, visitors=visitors.collect(state.visitors, visitors.bit(key)))` and persist;
- the frame render shows the visitor's quip (it takes precedence, like the memory quip) and passes
  `visitor_icon=visitors.glyph(key)`;
- a short motif (`pick_motif("greeting", tier)` is reused — calm and already gated against meetings)
  and a NeoPixel flash play once, in `main()`/the refresh block where `sound`/`pixels` are in scope
  and only when not in a meeting/sleep;
- `_maybe_journal` sets the `0b1000` flag (the visitor is decided before `_maybe_journal` runs at
  boot, so its sig gains a `visited_by` arg used only for the flag).

The chosen visitor is computed once per render decision and threaded into `_render_frame`
(`visitor=None` default). The keepsake bitmask is the permanent record; the visit is the moment.

## Data Flow

```
each cycle ─> visitors.eligible_visitors(oracle, season, tier) ─rare _choice roll─> pick_visitor ─> key|None
  on key ─> visitors.collect(state.visitors, bit) ─> state.visitors (NVM v5)
         ├─> _render_frame(visitor=key): quip = visitors.quip(key), visitor_icon = visitors.glyph(key)
         ├─> sound.play(motif) + pixels.flash(...) once (not in meeting/sleep)
         └─> _maybe_journal(visited_by=key): flags |= 0b1000 ─> generate_entry adds "a visitor came by"
```

## Error Handling / Offline-First

- Entirely on-device; no network. With no oracle and no season, condition-based visitors simply
  aren't eligible; the bond-based `cat` still can visit.
- The display glyph blit is guarded (missing `visitors.bmp` / bad index → no glyph, no crash), like
  the status tiles.
- The sound/pixel cue is best-effort (adapters may be `None`); a visit without hardware still shows
  the quip + glyph.
- A visitor may revisit; `collect` is idempotent (the keepsake bit is just re-set). The quip / glyph
  / journal line still occur — friends can come again.

## Testing

**Host pytest (pure):**
- `visitors.eligible_visitors`: each condition fires for its trigger (full moon → owl; rain → snail;
  winter → fox; bonded tier → cat; …); tolerates `None` oracle and `None` season; returns `()` when
  nothing matches.
- `visitors.pick_visitor`: returns the injected `choice` of the keys; `None` for empty.
- `visitors.collect`: sets the bit; idempotent; never clears other bits.
- `visitors.name`/`quip`/`glyph`/`bit`: round-trip the roster.
- `persistence` v5 pack/unpack round-trip (visitors preserved) + v4→v5 migration (visitors default 0)
  + v3/v2/v1 still migrate; `default_state().visitors == 0`; `evolve` carries `visitors`.
- `journal.generate_entry`: the `0b1000` flag yields a "visitor" closing line; absent → no such line.

**Device-only (AST-gated, on-device verified):** `code.py` visitor roll + cue + render wiring;
`display.py` corner glyph blit. `make_assets.py` runs and writes `assets/visitors.bmp` (assert the
file exists and has 8 tiles' worth of width).

**On-device:** over time, condition-matching days summon the right visitor occasionally; the corner
glyph + quip + cue appear; the journal notes the visit; the keepsake bitmask persists across reboots.

## Scope Guard (YAGNI)

- 8 fixed visitors, one byte. No visitor archive page, no trading, no rarity tiers, no per-visitor
  counts/stats, no visitor-specific sounds (reuse the greeting motif).
- A keepsake bit, a quip, a glyph, a cue, a journal line — nothing more.
- Only derived on-device conditions drive visitors; no network, no personal data.
